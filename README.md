## Standards

How Sneat repos are named and built. Start at the platform standards index:

- **[Platform standards index](https://github.com/sneat-co/backstage/blob/main/docs/standards/README.md)** (backstage) — one orientation page to all of the below.
- **[Repository naming + extension registry](https://github.com/sneat-co/sneat-specs/blob/main/standards/repo-naming.md)** (sneat-specs) — `sneat-*` platform, `<id>` extension impl (named products, no `ext-` prefix), `<id>-contract` contracts; discovery via GitHub topics.
- **[Frontend UX + screen-flow checklist](https://github.com/sneat-co/sneat-specs/blob/main/standards/frontend-ux/README.md)** (sneat-specs).
- **[Extension standards](https://github.com/sneat-co/sneat-libs/blob/main/docs/extension-standards/README.md)** (sneat-libs) — backend wiring · frontend apps · UX.
